# 🚀 CareerLog

> 취업 준비 과정을 한곳에서 관리하는 **FastAPI 기반 지원 관리 앱**

**CareerLog**는 여러 기업에 지원하면서 놓치기 쉬운 마감일, 지원 상태, 면접 기록, 자기소개서 문항, 진행 타임라인을 한 화면에서 정리할 수 있는 개인용 취업 준비 운영판입니다.

```text
지원 기업 관리 + D-Day + 면접 기록 + 자소서 문항 + 타임라인 + CSV 백업
```

## 🧩 주요 기능

### 🏢 지원 기업 관리

- ✅ 지원 기업 등록
- ✅ 지원 기업 목록 조회
- ✅ 지원 기업 상세 조회
- ✅ 지원 기업 수정 / 삭제
- ✅ 회사명, 직무, 마감일, 지원 상태, 메모 관리
- ✅ 지원 상태 빠른 변경

### 🔍 검색 · 필터 · 정렬

- 🔎 회사명, 직무, 메모 검색
- 🏷️ 지원 상태별 필터
- 📅 마감일 가까운 순 정렬
- 🆕 최근 등록 순 정렬
- 🔤 회사명 순 정렬

### ⏰ 마감일 D-Day

- 📌 `D-7`, `D-1`, `오늘 마감`, `D+3` 형태로 표시
- ⚠️ 마감 임박 항목 강조
- 📊 7일 내 마감 기업 수 대시보드 표시

### 📊 취업 준비 대시보드

- 📁 전체 지원 수
- ⏳ 7일 내 마감 수
- 🚨 마감 지난 지원 항목 수
- 🎤 전체 면접 기록 수
- 📈 상태별 지원 현황 집계

### 🎤 면접 기록 관리

- 🗓️ 면접일 저장
- 🧪 면접 유형 저장
- 📌 결과 / 상태 저장
- ❓ 면접 질문 기록
- 🧠 복기 메모 작성

### 📝 자기소개서 문항 관리

- 📋 기업별 자소서 문항 저장
- ✍️ 답변 초안 작성
- 🔄 작성 상태 관리
  - `초안`
  - `수정 중`
  - `제출 완료`
- 🔢 답변 글자 수 확인

### 🧭 지원 타임라인

- 📮 지원 완료
- 📄 서류 결과
- 🎤 면접 진행
- 🏁 최종 결과
- 🗒️ 기타 진행 메모

### 📦 CSV 백업

- 📤 지원 목록 CSV 내보내기
- 📥 CSV 파일로 지원 목록 가져오기
- 🧹 잘못된 상태값은 자동으로 `지원 예정` 처리

## 📦 CSV 가져오기 형식

CSV 파일은 아래 컬럼을 사용합니다.

```csv
company_name,position,deadline,status,memo
Example Corp,Backend Intern,2026-07-15,지원 예정,채용 공고 확인 필요
```

| 컬럼 | 필수 여부 | 설명 |
| --- | --- | --- |
| 🏢 `company_name` | 필수 | 회사명 |
| 💼 `position` | 필수 | 지원 직무 |
| 📅 `deadline` | 선택 | 마감일, `YYYY-MM-DD` 형식 |
| 🏷️ `status` | 선택 | 지원 상태 |
| 📝 `memo` | 선택 | 기업별 메모 |

사용 가능한 지원 상태:

```text
지원 예정, 지원 완료, 면접 예정, 면접 완료, 탈락, 합격
```

> `status`가 비어 있거나 지원하지 않는 값이면 자동으로 `지원 예정`으로 저장됩니다.

## 🗄️ 데이터베이스

서버를 처음 실행하면 프로젝트 루트에 `careerlog.db` SQLite 파일이 생성됩니다.

| 테이블 | 설명 |
| --- | --- |
| 🏢 `company_applications` | 지원 기업 기본 정보 |
| 🎤 `interview_records` | 기업별 면접 기록 |
| 📝 `cover_letter_questions` | 기업별 자기소개서 문항과 답변 |
| 🧭 `application_events` | 지원 진행 타임라인 |

> `careerlog.db`는 로컬 실행 데이터이므로 Git에는 포함하지 않습니다.
